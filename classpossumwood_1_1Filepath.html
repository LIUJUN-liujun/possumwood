Detailed Description

File path representation in the core of Possumwood. Consists of a "prefix", which is a variable with procedurally determined path (e.g., $NODES or $PLUGINS) and a relative path. Prefix can be empty - in that case, the path is an absolute path to a file. Conversions to a path are only necessary for file-browsing UIs - any actual file access should be implemented through IFilesystem abstraction to allow for mocking.
